# Bugs And Watchlist

## Исправлено в предрелизе 2026-06-25

- Опубликованный JSON-отчет больше не теряет локальные изменения после обновления страницы.
- Статус отчета больше не добавляется внутрь самого отчета: он должен жить только на главной странице.
- Главная различает `Опубликованный отчет`, `Есть изменения` и `Локальный черновик`.
- Неделя 25 в предрелизной сборке берется из `weekly_workboard_week-25_2026-06-18.html`, а не из старой недели 22.
- `report.html` больше не показывает старую неделю в статической шапке при ошибке загрузки данных.
- Ближайшие ключевые результаты больше не должны тянуть завершенные проекты.
- Завершенные проекты не смешиваются с активным dashboard: свежие показываются в результатах недели, старые уходят в архив завершенных проектов.
- `paused` больше не делает направление проблемным автоматически.
- Текст в `Риски / Фокус CEO` считается сигналом внимания, но один дальний риск не должен сразу красить все направление в проблему.
- Оценка прогресса теперь смотрит на близость даты релиза, а не на старую стартовую дату задачи.

## Смотреть внимательно

- Если открыть `report.html` напрямую без параметра `?data=...`, он может показать ошибку данных. Нормальная ссылка должна идти из `reports.json`.
- Если браузер уже хранил локальные изменения, главная честно покажет `Есть изменения`. Это не серверный статус, а локальная копия в текущем браузере.
- После загрузки на сервер GitHub Pages может отдавать старый кэш. Для проверки лучше открывать страницу с параметром версии.
- Старый рабочий проект и предрелиз могут отличаться по логике. Не смешивать файлы между ними вручную без проверки.

## Исправлено

- Опубликованные отчеты из `weekly-reports/reports.json` теперь должны иметь приоритет над локальными дублями той же недели.
- Локальные отчеты, совпадающие с опубликованной неделей, не должны отображаться на главной.
- Для опубликованных отчетов дата из `updatedAt` показывается как дата отчета, а не как время локального обновления.
- Кнопка “Удалить” не должна показываться у опубликованных отчетов.

## Проверено

- `weekly-reports/reports.json` доступен на GitHub Pages.
- Неделя 22 отображается как “Опубликованный отчет”.
- У опубликованных недель нет кнопки “Удалить”.

## Наблюдения

- GitHub Pages может отдавать старый `index.html` из кэша. После публикации лучше открывать страницу с параметром версии.
- Если `reports.json` не загружается, главная видит только локальные отчеты из `localStorage`.
- Если в браузере есть старые локальные отчеты, они могут оставаться в `localStorage`, но не должны перекрывать опубликованные недели.
- `demo-page-main/demo-page-main` хранит скачанную GitHub-копию и не является местом для рабочих правок.

## Риски

- Проект работает без backend, поэтому публикация зависит от ручной загрузки HTML-отчета и `reports.json`.
- Если загрузить HTML-отчет, но не обновить `reports.json`, отчет не появится в общем списке на главной.
- Если загрузить новый `index.html`, но открыть старую кэшированную страницу, фикс визуально не проявится.
- Если ответственный не нажмет “Принять изменения” после импорта patch, следующий patch может чаще показывать конфликты.

## Проверить вручную позже

- Импорт нескольких patch-файлов подряд.
- Конфликтный импорт одного и того же поля.
- Скачивание финального HTML после нескольких импортов.
- Локальный отчет новой неопубликованной недели отображается как локальный.
- PNG-экспорт на GitHub Pages.
- Поведение на мобильном экране.
