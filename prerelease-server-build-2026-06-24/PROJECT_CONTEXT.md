# Project Context

Короткий файл для нового чата Codex: с него начинать, если нужно быстро понять проект без пересказа всей истории.

## Что это

Проект Healbe Release Hub: статический HTML-инструмент без backend для:

- недельных продуктовых отчетов;
- продуктовых направлений;
- дорожных карт;
- подготовки файлов для GitHub Pages.

Пользователь не программист, поэтому ответы нужны короткие, без лишнего кода. Если решение выглядит неправильным, с пользователем можно и нужно спорить.

## Главное правило папок

`demo-page-main` и похожие скачанные папки с GitHub/сервера считаются архивом текущего рабочего состояния.

Их нельзя менять. Можно только читать, сравнивать и брать оттуда исходники.

Рабочие правки делать в корне проекта или в отдельной тестовой сборке.

## Текущая стратегия переезда

Старый рабочий проект остается основным.

Новая версия тестируется отдельно в папке:

`prerelease-server-build-2026-06-24`

На старой главной есть кнопка `Новый формат`, которая ведет в эту папку. Старую главную не переводим на новую модульную схему, чтобы она не зависела от корневых `assets`.

## Что заливать для старой главной

Для мягкого входа в новую версию собран hotfix:

`server-main-hotfix-2026-06-25`

Внутри только автономный `index.html`.

Его можно залить в корень сервера вместо старого `index.html`. Папку `assets` для старой главной заливать не нужно.

## Что заливать для новой версии

Для обновления новой тестовой версии заливать измененные файлы внутри:

`prerelease-server-build-2026-06-24`

Чаще всего меняется:

- `prerelease-server-build-2026-06-24/assets/home.js`;
- `prerelease-server-build-2026-06-24/assets/home.css`;
- `prerelease-server-build-2026-06-24/assets/weekly-workboard.js`;
- `prerelease-server-build-2026-06-24/assets/weekly-workboard.css`;
- `prerelease-server-build-2026-06-24/weekly-reports/data/*.json`;
- `prerelease-server-build-2026-06-24/weekly-reports/reports.json`.

## Последняя важная логика

- Локальный черновик новой недели должен показываться выше опубликованных старых недель, если он новее по дате.
- Опубликованный отчет скрывает только локальный дубль той же недели.
- При создании новой недели очищаются недельные поля:
  - `artifactNote` / “Сделано”;
  - `focus` / “Риски / Фокус CEO”;
  - `ceoFocus`.
- Завершенные задачи из прошлой недели при создании новой недели должны уходить в архив завершенных проектов, а не в “Завершено на этой неделе”.
- Статус отчета показывается на главной, а не внутри самого отчета.

## Проверка

Не использовать Edge headless как основной способ проверки.

Стандартный способ:

1. Запустить локальный сервер:

   `powershell -ExecutionPolicy Bypass -File tools\serve-local-preview.ps1 -Port 5317`

2. Открывать через встроенный браузер Codex:

   `http://127.0.0.1:5317/`

`file://` для проверки новой версии не использовать.
